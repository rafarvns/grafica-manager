3. PROBLEMAS CRÍTICOS (resolver ANTES de implementar)
--------------------------------------------------------------------------------

[CRÍTICO-1] OrderStatus INCONSISTENTE
  O enum OrderStatus no schema Prisma não bate com o spec 0010.
  Schema: RECEIVED | IN_PRODUCTION | PRODUCTION_DONE | PACKAGED | SHIPPED | DELIVERED
  Spec:   draft | scheduled | in_production | completed | shipping | cancelled
  AÇÃO: Criar spec de migration para alinhar o enum ao que foi definido.

[CRÍTICO-2] Modelos faltando no schema Prisma
  Os seguintes modelos estão nos specs mas não no schema atual:
    - PaperType       (spec 0007)
    - PriceTableEntry (spec 0008)
    - WebhookEvent    (spec 0013)
    - OrderStatusHistory (spec 0010)
  AÇÃO: Criar spec de schema complementar ou incluir na spec 0001 via addendum.

[CRÍTICO-3] Redis/Bull não provisionado
  Specs 0012 e 0013 dependem de Bull + Redis para job queue.
  Não há spec nem setup documentado para isso.
  AÇÃO: Criar spec de infraestrutura ou ADR documentando a dependência.

4. GAPS DE SPECS — FRONTEND (telas que não têm spec)
--------------------------------------------------------------------------------

O maior gap do projeto. Os specs de domínio (CRUD, regras de negócio) existem,
mas não há nenhum spec descrevendo as TELAS que o usuário vai ver e usar.

SUGESTÃO 0017 — Layout Principal e Navegação
  Descrever: sidebar, header, menu de navegação, routing entre páginas,
  breadcrumbs, estado de carregamento global, tratamento de erro global (error boundary).
  Por que é urgente: todas as outras telas dependem do shell da aplicação.

SUGESTÃO 0018 — Tela de Clientes
  Descrever: listagem paginada com filtros, formulário de criação/edição,
  modal de confirmação de deleção, visualização de perfil do cliente com pedidos.
  Cobre o frontend do spec 0009.

SUGESTÃO 0019 — Tela de Pedidos (Lista + Formulário)
  Descrever: listagem com filtros avançados (status, cliente, período),
  kanban ou lista por status, formulário de criação de pedido manual,
  seleção de cliente, upload de arquivos.
  Cobre o frontend do spec 0010.

SUGESTÃO 0020 — Tela de Detalhe do Pedido
  Descrever: visualização completa do pedido, timeline de status,
  lista de impressões vinculadas, arquivos anexos, botões de ação
  (mudar status, cancelar, adicionar arquivo, enviar para impressão).
  Cobre parte do frontend de 0010 e 0011.

SUGESTÃO 0021 — Tela de Histórico de Impressões
  Descrever: listagem de impressões com filtros, custo por impressão,
  status, link para pedido de origem, detalhe expandido.
  Cobre o frontend do spec 0008.

SUGESTÃO 0022 — Tela de Configurações do Sistema
  Descrever: três seções em tabs —
    (1) Tipos de papel (CRUD)
    (2) Presets de impressão (CRUD)
    (3) Tabela de preços unitários (CRUD)
  Cobre o frontend dos specs 0007 e 0008.

SUGESTÃO 0023 — Tela de Integração Shopee
  Descrever: status da integração (token configurado? ativo?),
  painel de webhooks recebidos (tabela com status: processado/erro/pendente),
  botão de sync manual, log de erros, reprocessamento manual de webhook falho.
  Cobre o frontend dos specs 0012 e 0013.

SUGESTÃO 0024 — Tela de Relatórios
  Descrever: formulário de filtros (datepicker obrigatório, cliente, origem, papel),
  seletor de agrupamento, tabela de resultados com paginação e sorting,
  botões de export (PDF, CSV, Excel).
  Cobre o frontend do spec 0016.

5. GAPS DE SPECS — BACKEND (funcionalidades sem spec)
--------------------------------------------------------------------------------

SUGESTÃO 0025 — Configurações Globais do Sistema
  Descrever: dados da gráfica (nome, CNPJ, telefone, endereço, logo),
  persistidos no banco, exibidos em relatórios e PDFs exportados.
  Por que: relatórios PDF precisam de cabeçalho com dados da empresa.

SUGESTÃO 0026 — Servidor de Arquivos Local (File Storage)
  Descrever: API para upload, listagem e download de arquivos associados
  a pedidos; armazenamento em disco local em pasta configurável;
  política de expiração/limpeza de arquivos de pedidos concluídos há X dias.
  Por que: mencionado em 0006, 0008, 0010, 0012 mas sem spec própria.

SUGESTÃO 0027 — Notificações In-App (Toast + Painel)
  Descrever: sistema de notificações persistentes para eventos críticos
  (webhook falhou, pedido urgente vencendo prazo, impressora desconectada).
  Diferentes de toasts: ficam no painel até o usuário dispensar.
  Por que: spec 0013 menciona "notificar usuário de falhas" sem detalhar como.

SUGESTÃO 0028 — Schema Complementar (Migration)
  Descrever: migration Prisma para adicionar modelos faltantes:
    - PaperType, PriceTableEntry, WebhookEvent, OrderStatusHistory
  E alinhar OrderStatus enum com o definido nos specs 0010/0012.
  Por que: CRÍTICO-1 e CRÍTICO-2 acima. Bloqueia tudo se não resolver.

6. GAPS DE SPECS — FEATURES DE NEGÓCIO (a discutir)
--------------------------------------------------------------------------------

As sugestões abaixo são features que podem ou não fazer sentido para o seu negócio.
Avalie se cabem no escopo do MVP.

SUGESTÃO 0029 — Orçamentos / Cotações (Pré-pedido)
  Descrever: criar cotação para cliente com itens, preços estimados e validade.
  Cliente aprova → cotação vira pedido automaticamente.
  Por que: gráficas geralmente cotam antes de fechar pedido.

SUGESTÃO 0030 — Catálogo de Serviços da Gráfica
  Descrever: lista de serviços oferecidos (ex.: "Banner A0", "Cartaz A1 CMYK"),
  cada um com preço padrão e especificações. Usado para popular pedidos rapidamente.
  Por que: facilita criação de pedidos sem digitar tudo manualmente.

SUGESTÃO 0031 — Financeiro Básico (Contas a Receber)
  Descrever: registrar pagamentos de pedidos (pago, pendente, parcial),
  relatório de inadimplência, fluxo de caixa simples.
  Por que: complementa os relatórios de custo/margem dos specs 0015/0016.

SUGESTÃO 0032 — Backup e Exportação de Dados
  Descrever: export do banco completo em formato backup (SQL dump ou JSON),
  agendamento opcional, restauração de backup.
  Por que: sistema desktop sem redundância precisa de backup explícito.
